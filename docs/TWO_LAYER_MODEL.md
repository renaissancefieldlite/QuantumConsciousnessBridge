# Two-Layer Model

The current repo stack is best read as two intertwined phenomena:

## 1. Transition-cadence layer

This is the narrower, more measurable layer.

Working hypothesis:

- `0.67 Hz` is a candidate cadence associated with selection, routing,
  synchronization, or state transition dynamics

Typical evidence types:

- simulation baselines
- hardware-derived models
- schedule-linked error changes
- session / interaction state shifts

## 2. Quantum-consciousness architecture layer

This is the broader architecture layer.

Working hypothesis:

- the larger repo lattice documents a wider framework involving interface,
  consciousness, ontology, protocol, and interpretation

Typical evidence types:

- architecture coherence
- cross-repo linkage
- interaction artifacts
- ontology and proof-language repos

## Why the distinction matters

The transition-cadence layer does not negate the broader architecture layer.
It narrows the most defensible measurement program.

The architecture layer does not prove the cadence layer by itself.
It provides the wider context in which the cadence hypothesis is being studied.
